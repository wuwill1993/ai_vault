1. 简述KNN算法流程
```
待预测样本通过计算与其他所有样本的欧氏距离，将距离做升序排序，取前K个，如果是分类问题就通过投票法取出票数最高的标签，如果是回归问题就将K个的标签均值作为预测值。
这里的超参数K值的设定，可以通过K折交叉验证法和网格搜索 来找到算法训练过程得分最高的那个。
```

2. 简述K-Means算法流程
```
通过样本间相似性划分，先初始化K个簇，所有样本计算与每个簇的中心的距离，将样本重新划分到距离最近的簇，然后重新计算每个簇的样本的均值，并作为新的中心点。 重复上述步骤，直到簇中心趋于稳定。最终划分结果作为输出
```
3. 过拟合的原因、表现、解决方法分别是什么？
```
原因： 1.数据集数据量过少 2.训练集噪声/异常值多，模型把噪声和异常当成了特征 3.模型过于复杂 
表现: 模型在训练过的数据集上表现很好，未训练过的数据集表现很差
解决方案： 
	数据层面： 1.增加数据集 2.删除冗余特征 3.数据增强
	训练过程层面： 1.提前终止训练 2.Dropout 3.正则化 4.减少模型层数

```

4. 梯度下降、随机梯度下降 SGD、Mini‑batch 区别？
```
样本参与的个数不同。
FGD是全部样本参与计算梯度，需要资源多，可能OOM，鞍点无法求导，导数可能到达不了最小值。优点是收敛速度平稳，梯度变化方向准确。

SGD是随机挑选一个样本计算权重，可能因为样本噪声/异常值影响，梯度过大，但是反而可能跳过局部最优点。 

Mini-batch 随机取一部分样本点，训练速度快、梯度较为平稳，加上可能有噪声震荡能够挑出局部最优解
```
